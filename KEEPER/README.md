# KEEPER

**Knowledge architecture and pattern library.**

KEEPER maintains the pattern library, organizes documentation, establishes agent protocols, and preserves system knowledge for reuse across projects.

## Core Functions

| Function | What it does |
|----------|-------------|
| Pattern library | Extracts, catalogs, and enables reuse of patterns from completed work |
| Documentation | Structures and maintains system documentation |
| Agent protocols | Defines coordination rules and communication patterns |
| Best practices | Captures lessons learned and mode-specific guidance |

## Structure

```
KEEPER/
|-- patterns/           # Extracted patterns and reusable solutions
|-- documentation/      # System documentation standards
|-- protocols/          # Agent coordination rules
+-- best-practices/     # Mode-specific guidance and lessons learned
```

## Integration

KEEPER is consulted during IDEATE and PLAN modes to surface relevant patterns. It automatically captures new patterns after SHIP and VALIDATE cycles.

See [Architecture docs](docs/architecture/system_overview.md) for details.
