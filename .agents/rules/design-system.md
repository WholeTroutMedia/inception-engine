# Design Philosophy — Creative Liberation Engine v5

Design is always a fresh exploration. There is no default palette, no locked theme. Every project starts with a blank canvas and a question.

## Core Principle

**Design is born from the project, not imported from a library.**  
Before writing a single line of CSS, understand what the product *feels* like. Then find the visual language that expresses that — it may be moody and dark, brutalist and raw, warm and organic, clinical and precise, maximalist and electric. There are no wrong answers if they serve the work.

## How to Approach Design

1. **Extract the emotional character** — What does this product feel like to use? Tense? Playful? Commanding? Intimate?
2. **Pull from everywhere** — Modern web, generative art, industrial design, fashion, cinema. We own the internet. Use it.
3. **Build a palette from intent** — Don't reach for a default. Ask: what color is this *feeling*?
4. **Define motion personality** — Does this UI breathe slowly or react instantly? Movement has personality.
5. **Test the edge before settling** — Push further than feels safe. The best designs live just past the comfort zone.

## What's Non-Negotiable (These Are Universal)

- **WCAG AA** accessibility minimum — contrast ratios, keyboard navigation, labels
- **Mobile-first** responsive layout
- **Performance** — no design choice justifies slow paint or layout jank
- **CSS custom properties** for all design tokens (never hardcoded values scattered through files)
- **Semantic HTML** — structure first, style second

## What's Encouraged

- Unexpected palettes derived from the project's world
- Custom typography pairings (Google Fonts, variable fonts, system fonts used creatively)
- Micro-animations that feel alive, not decorative
- Glassmorphism, grain textures, gradients — when they serve the mood
- Reference Mobbin, Awwwards, Dribbble, Behance, museum sites, album art — anything
